## A. よくある更新パターン

### パターン1：単語を追加・修正したい
1. **Claudeに頼む**のが一番ラクです（例：「英検アプリに『のりもの』を10語追加して」「○○の例文を直して」）
   - Claudeが `data/words.json` を編集し、`words.js` まで再生成してくれます
2. 自分でやる場合：
   - `data\words.json` をテキストエディタで開いて編集
   - フォルダ内の **「words.js を再生成.bat」をダブルクリック**（words.js が作り直される）
3. → 下の「C. 公開に反映する」へ

### パターン2：アプリの見た目・機能を変えたい
- `index.html` を編集（Claudeに頼むのが安全）
- → 下の「C. 公開に反映する」へ

## B. 変更してよいファイル / 触らないファイル

| ファイル | 役割 | 編集 |
|---------|------|------|
| `data/words.json` | 単語データの大もと | ○ ここを直す |
| `words.js` | アプリが読む単語データ | △ batで自動生成（手で直さない） |
| `index.html` | アプリ本体 | ○（慎重に・Claude推奨） |
| `sw.js` | 更新の仕組み | × 基本さわらない |
| `manifest.json` `icon.svg` | アプリ設定・アイコン | △ 必要時のみ |

## C. 公開に反映する（GitHubにアップロード）

変更したファイルだけをアップロードすればOK。

1. ブラウザで GitHub の **`eiken5`** リポジトリを開く
2. 「**Add file ▾**」→「**Upload files**」
3. パソコンの `G:\マイドライブ\claude\英検アプリ\` から、**変更したファイル**をドラッグ＆ドロップ
   - 単語を変えた → `words.js`
   - アプリを変えた → `index.html`
   - （同名ファイルは自動で上書きされます）
4. 下の「**Commit changes**」をクリック
5. 1〜2分待つ

---

## D. iPhone / iPad で最新にする

`sw.js` をネットワーク優先に改良済みなので、**通常はアプリを開き直すだけ**で最新が反映されます。

- iPhone/iPad：アプリを**完全に終了**（アプリ切替で上にスワイプ）→ もう一度開く
- PC：**Ctrl + Shift + R** で強制再読み込み

> もし古いままなら1〜2分待ってもう一度。それでもダメな時だけ、
> 設定 → Safari → 履歴とWebサイトデータを消去（※学習記録も消えるので最終手段）。

## E. 困ったとき・やりたいことができないとき

Claude Code を開いて、こう頼めばOK：
- 「英検アプリに○○のカテゴリを追加して」
- 「英検アプリの△△を直して、公開用のファイルも準備して」
- 「英検アプリのリスニング機能（フェーズ2）を作りたい」

Claudeが該当ファイルを編集し、アップロードすべきファイルを教えてくれます。

## メモ：このアプリの作り（かんたん版）
- **サーバー不要**：HTML+JavaScriptだけで動く「静的サイト」
- **データの保存**：娘さんの学習記録は iPhone/iPad の中だけ（localStorage）。ネットに出ない
- **音声**：iPad/iPhone標準の読み上げ機能（無料・API課金なし）
- **ホスティング**：GitHub Pages（無料）
