## C. 公開に反映する（git push）

### いちばんラク：Claudeに頼む
Claude Code で「**英検アプリの変更を公開して**」と言えば、変更の保存（コミット）と公開（プッシュ）をまとめてやってくれます。1〜2分でライブURLに反映されます。

### 自分でやる場合（コマンド）
Claude Code か PowerShell で、次の1行を実行するだけです：

```powershell
git -C "G:\マイドライブ\claude\英検アプリ" add . ; git -C "G:\マイドライブ\claude\英検アプリ" commit -m "更新" ; git -C "G:\マイドライブ\claude\英検アプリ" push
```

- `add .` … 変更したファイルをぜんぶ対象にする
- `commit -m "更新"` … 変更を1つの記録にまとめる（""の中は内容メモ。何でもOK）
- `push` … GitHubへ送る（＝公開に反映）

> **メモ：** 昔のように「GitHubサイトでUpload files」する必要はもうありません。もし間違えて両方やっても壊れませんが、git push に一本化するのがおすすめです。

## D. iPhone / iPad で最新にする

`sw.js` をネットワーク優先に改良済みなので、**通常はアプリを開き直すだけ**で最新が反映されます。

- iPhone/iPad：アプリを**完全に終了**（アプリ切替で上にスワイプ）→ もう一度開く
- PC：**Ctrl + Shift + R** で強制再読み込み

> もし古いままなら1〜2分待ってもう一度。それでもダメな時だけ、
> 設定 → Safari → 履歴とWebサイトデータを消去（※学習記録も消えるので最終手段）。

## メモ：このアプリの作り（かんたん版）
- **サーバー不要**：HTML+JavaScriptだけで動く「静的サイト」
- **データの保存**：娘さんの学習記録は iPhone/iPad の中だけ（localStorage）。ネットに出ない
- **音声**：iPad/iPhone標準の読み上げ機能（無料・API課金なし）
- **ホスティング**：GitHub Pages（無料）
- **公開のしくみ**：ローカルで編集 → `git push` → GitHubが自動でサイトを更新（2026-06-04にこの方式へ移行）
